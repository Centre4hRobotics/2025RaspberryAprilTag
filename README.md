# 2025RaspberryAprilTag

![Status Name](https://github.com/Centre4hRobotics/2025RaspberryAprilTag/actions/workflows/main.yml/badge.svg)

This repository holds the source code for the Raspberry Pi-based AprilTag system for CC4H Robotics. Visit [learn.cc4hrobotics.org/programming/raspberry-pi](https://learn.cc4hrobotics.org/programming/raspberry-pi/) for background information and tutorials.

Each different camera, meaning each camera with a different sensor or lens, should get its own profile in `config/CameraProfiles.json`.
Each physical camera should have its own entry in `config/Settings.json`
> Currently, multiple physical cameras are not supported.

## Setup

To set up the virtual environment, run `setup.sh`. It will create the virtual environment. The script assumes that it is being run on a system with Photonvision, and that the repo is in the home directory. If that is not true, you'll need to edit `/etc/systemd/system/rpivision.service` and change the path to `run.sh` to wherever it actually is.

## Running

To run the code, either use the systemd service, or run `run.sh` in a terminal. You will need to disable Photonvision or any other software using the cameras.
> If it doesn't work normally, you may also have to run `run.sh` with root permissions. Try `sudo ./run.sh`.

## Calibrating

Run `manual_camera.py` to take pictures of a ChArUco board. It publishes the current image it can see to network tables, but you need to use the command line for actually taking the image. Then, run `calibrate.py` (in `calibration/`) to generate a camera profile. It does not include framerate or rotation, so you'll need to add that too.
> Both of these require the virtual environment generated by `setup.sh`.
